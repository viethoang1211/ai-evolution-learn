Từ prompt-based usage sang skill-based execution
1. Tư duy gốc: Skills không phải “mẹo prompt”, mà là tài sản vận hành
Phần lớn mọi người đang dùng AI theo kiểu:
• hỏi
• nhận câu trả lời
• chỉnh lại prompt
• thử lại
Cách này phù hợp cho tác vụ ngắn, đơn lẻ, ít rủi ro.
Nhưng khi công việc bắt đầu có:
• nhiều bước
• nhiều nguồn dữ liệu
• nhiều tiêu chí đúng/sai
• yêu cầu tái sử dụng
• yêu cầu kiểm chứng
• yêu cầu mở rộng cho team
thì prompt không còn đủ nữa.
Lúc đó, thứ cần xây không phải là “prompt tốt hơn”, mà là một hệ skill có cấu trúc, nơi agent không chỉ đọc hướng dẫn mà còn:
• hiểu ngữ cảnh
• gọi đúng tri thức
• chạy đúng quy trình
• dùng đúng script
• kiểm tra đầu ra
• tự cải thiện sau mỗi lỗi
Nói cách khác:
Prompt là câu lệnh tạm thời.
Skill là năng lực đóng gói.
Skill system là bộ nhớ vận hành của cá nhân hoặc tổ chức.
2. Mô hình nền tảng: 5 lớp của một hệ Skill hoàn chỉnh
Tôi đề xuất mô hình 5 lớp để áp dụng cho cá nhân và dự án:
Lớp 1: Intent Layer - Mục tiêu cần đạt
Đây là lớp xác định:
• bài toán là gì
• đầu ra nào được xem là hoàn thành
• tiêu chuẩn đúng/sai là gì
• phạm vi được phép xử lý đến đâu
Nếu không rõ lớp này, AI sẽ sinh nhiều thứ “có vẻ đúng” nhưng không giải quyết đúng việc.
Ví dụ:
• Viết proposal đào tạo cho doanh nghiệp
• Tạo outline khóa học cho nhóm sale
• Phân tích use case AI cho phòng HR
• Tạo landing page outline cho sản phẩm AI
• Sinh code module upload file có kiểm thử
Intent Layer trả lời: Ta đang muốn hoàn thành điều gì, trong tiêu chuẩn nào?
Lớp 2: Knowledge Layer - Tri thức để hành động
Đây là nơi chứa:
• domain knowledge
• business rules
• best practices
• style guide
• API references
• framework nội bộ
• brand voice
• template chuẩn
Ví dụ với cá nhân:
• phong cách viết bài LinkedIn của bạn
• framework đào tạo AI cho doanh nghiệp
• mẫu proposal chuẩn enterprise
• checklist xây system prompt
• tiêu chí đánh giá use case AI
Ví dụ với dự án:
• tài liệu sản phẩm
• tài liệu database
• API docs
• quy tắc coding
• quy trình release
• sơ đồ hệ thống
Knowledge Layer trả lời: Muốn làm đúng thì cần biết gì?
Lớp 3: Execution Layer - Cách thực thi
Đây là phần nhiều người bỏ qua nhất.
Skill không nên chỉ nói “làm thế nào”, mà còn nên chứa thứ agent có thể dùng trực tiếp:
• script
• command
• workflow
• query mẫu
• file template
• automation
• scaffold code
• checklist thao tác
Ví dụ:
• file script để tạo cấu trúc dự án
• template PRD
• command deploy
• SQL query kiểm tra data
• prompt-template để generate case study
• Python script parse file CSV
• template agenda cho workshop
Execution Layer trả lời: Làm bằng cách nào, theo từng bước có thể chạy được?
Lớp 4: Verification Layer - Kiểm chứng trước khi xong
Đây là lớp quan trọng nhất.
Rất nhiều hệ AI thất bại không phải vì “không sinh ra được”, mà vì:
• không test
• không đối chiếu
• không mô phỏng
• không xác thực với thực tế
• không có tiêu chí pass/fail rõ ràng
Một skill mạnh phải có:
• expected output
• test cases
• anti-patterns
• checklist kiểm tra
• simulation steps
• sanity checks
• compare expected vs actual
Ví dụ:
• Proposal có đủ các phần bắt buộc chưa?
• Code có chạy test không?
• Nội dung đào tạo có đúng đối tượng non-tech không?
• Prompt có tạo đầu ra theo đúng format không?
• Agent có xử lý được edge case khi input thiếu không?
Verification Layer trả lời: Làm sao biết thứ vừa tạo ra là đúng, dùng được, và đáng tin?
Lớp 5: Evolution Layer - Học từ lỗi và nâng cấp skill
Skill không phải tài liệu bất động.
Mỗi lần xảy ra:
• lỗi
• hallucination
• output sai format
• xử lý thiếu bước
• hiểu sai business rule
• vấp edge case
• phải sửa tay nhiều
thì đó là dữ liệu để cập nhật skill.
Evolution Layer gồm:
• gotchas.md
• failure cases
• edge cases
• lessons learned
• changelog
• version history
• improvement notes
Evolution Layer trả lời: Sau mỗi lần dùng, hệ skill thông minh hơn ở đâu?

4. Nguyên tắc vàng khi thiết kế skill
Nguyên tắc 1: Một skill = một nhiệm vụ duy nhất
Không nhồi nhiều việc vào một skill.
Sai:
• “skill_viết_bài_kiểm_tra_đăng_facebook_tạo_ảnh”
Đúng:
• write-facebook-post
• verify-facebook-post
• generate-cover-image-brief
• publish-social-content
Một skill càng đơn nhiệm, agent càng dễ chọn đúng và dùng đúng.
Nguyên tắc 2: Skill là folder, không phải chỉ là 1 file
Một folder skill có thể chứa:
• hướng dẫn
• examples
• scripts
• templates
• test cases
• data mẫu
• changelog
Điều này quan trọng vì context tốt không đến từ một đoạn text dài, mà đến từ cấu trúc rõ.
Ví dụ:
skills/
write-enterprise-proposal/
SKILL.md
templates/
proposal-outline.md
pricing-table.md
examples/
sample-proposal-1.md
verification/
checklist.md
assets/
company-profile-summary.md
changelog.md
Skill như vậy mạnh hơn rất nhiều so với một prompt dài 500 dòng.
Nguyên tắc 3: Luôn có verification đi kèm
Không có verification thì skill chỉ là generator.
Muốn skill trở thành năng lực thực thi đáng tin, phải có:
• checklist
• tiêu chí đạt
• test case
• lỗi thường gặp
• cách đối chiếu 
Nguyên tắc 4: Cấu trúc tốt hơn kiểm soát cứng
Sai lầm lớn là cố chỉ đạo agent từng bước nhỏ:
• bước 1 làm cái này
• bước 2 làm cái kia
• bước 3 không được lệch
• bước 4 bắt buộc câu này
• bước 5 không được khác
Cách này làm agent bị “ngộp”, thiếu linh hoạt, xử lý kém khi gặp biến thể.
Thay vào đó:
• xác định mục tiêu
• cung cấp context chuẩn
• đưa công cụ phù hợp
• định nghĩa tiêu chuẩn xác minh
• cho agent đủ không gian để thích nghi
Tư duy đúng là:
Đừng cố điều khiển từng cử động của AI.
Hãy thiết kế môi trường để AI hành động tốt.

8. Mẫu SKILL.md chuẩn
# SKILL: write-enterprise-proposal
## Purpose
Tạo proposal đào tạo/tư vấn chuẩn enterprise, rõ logic, phù hợp lãnh đạo doanh nghiệp.
## Use When
- Cần viết proposal cho khách hàng doanh nghiệp
- Cần chuẩn hóa cấu trúc đề xuất
- Cần giữ phong cách chuyên nghiệp, chiến lược
## Required Inputs
- Tên khách hàng
- Bối cảnh / nhu cầu
- Đối tượng học viên
- Mục tiêu chương trình
- Thời lượng dự kiến
- Yêu cầu đặc thù
## Expected Output
- Proposal có cấu trúc hoàn chỉnh
- Ngôn ngữ rõ, thuyết phục, không quá học thuật
- Có logic: bối cảnh -> vấn đề -> mục tiêu -> chương trình -> phương pháp -> kết quả kỳ vọng
## Execution Approach
1. Phân tích bối cảnh doanh nghiệp
2. Nhóm nhu cầu thành 3–5 trục chính
3. Thiết kế chương trình theo mức độ phù hợp đối tượng
4. Viết proposal theo cấu trúc chuẩn
5. Kiểm tra độ rõ, tính khả thi, tính thuyết phục
## Quality Criteria
- Không lan man
- Đúng ngữ cảnh ngành
- Phù hợp đối tượng non-tech nếu được chỉ định
- Có outcome cụ thể
- Không dùng ngôn ngữ quá chung chung
## Verification
- Có đủ các phần bắt buộc chưa?
- Mỗi phần có gắn với nhu cầu thực tế không?
- Đề xuất có logic triển khai không?
- Có nội dung nào quá generic không?
## Edge Cases
- Khách hàng mô tả nhu cầu mơ hồ
- Yêu cầu quá rộng so với thời lượng
- Đối tượng học viên rất không đồng đều
- Muốn “thực chiến” nhưng chưa có dữ liệu thực tế
## References
- templates/proposal-outline.md
- examples/sample-proposal-b2b.md
- verification/checklist.md
## Changelog
- v1.2: thêm phần outcome theo phòng ban
- v1.3: thêm kiểm tra độ phù hợp non-tech 
9, Cơ chế verification: phần bắt buộc trong mọi skill
Tôi đề xuất framework kiểm chứng 4C:
. Correctness: Có đúng không?
• Đúng logic?
• Đúng dữ kiện?
• Đúng quy trình?
• Đúng format?
. Completeness: Có đủ không?
• Đủ phần chưa?
• Đủ đầu vào chưa?
• Có bỏ sót bước quan trọng không?
. Context-fit: Có hợp ngữ cảnh không?
• Đúng ngành chưa?
• Đúng đối tượng chưa?
• Đúng mục tiêu chưa?
• Đúng mức độ kỹ thuật chưa?
. Consequence: Dùng thật có ổn không?
• Nếu đem đi gửi khách hàng thì có vấn đề gì?
• Nếu chạy script thật có gây lỗi gì?
• Nếu giao cho team thì có hiểu sai không?
Mọi skill đều nên đi kèm câu hỏi cuối:
Nếu kết quả này được dùng ngay trong thực tế, rủi ro lớn nhất là gì?
Đây là cách nâng chất lượng từ “có vẻ ổn” lên “đủ tin cậy để dùng”.

12. Công thức vận hành ngắn gọn
Có thể nhớ hệ này bằng công thức:
SCOPE → SKILL → EXECUTE → VERIFY → EVOLVE
SCOPE
Xác định bài toán, mục tiêu, đầu ra
SKILL
Chọn hoặc tạo module phù hợp
EXECUTE
Cho agent thực thi với tri thức + công cụ + workflow
VERIFY
Kiểm tra bằng checklist, test, đối chiếu
EVOLVE
Cập nhật skill từ lỗi và edge case
Đây là vòng lặp tạo ra năng lực tích lũy.
Skill system giúp bạn mở rộng năng lực cá nhân và tổ chức!
